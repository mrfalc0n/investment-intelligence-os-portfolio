# Public Sanitization Standard

## TL;DR

This repository publishes architecture and synthetic examples only. It intentionally excludes private financial data, broker details, credentials, and actionable trade instructions.

## Publishable

- Generalized architecture diagrams.
- Synthetic holdings and candidate examples.
- Sanitized workflow packets.
- Agent responsibility descriptions.
- Safety and review-gate rules.
- Account-domain reasoning patterns.

## Not Publishable

- Real holdings, balances, P/L, account names, account numbers, or tax lots.
- Broker credentials, API keys, access tokens, session files, or OAuth artifacts.
- Notion database IDs, private dashboard URLs, or raw workspace exports.
- Real scanner outputs, watchlists, action queue rows, or open-order states.
- Trade instructions that someone could copy.
- Personalized financial advice.

## Sanitization Checklist

Before publishing material here:

1. Replace holdings, tickers, account names, and dollar amounts with synthetic examples.
2. Remove all credentials, IDs, private paths, and live service references.
3. Convert execution commands into architecture descriptions.
4. Keep broker integration language read-only and safety-focused.
5. Mark examples as synthetic.
6. Avoid current market opinions or actionable recommendations.

## Portfolio Rule

The public repo should show systems thinking, not portfolio intent.
